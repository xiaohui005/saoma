# 悬浮启动按钮设计

## 目标

在保留现有后台待命通知入口的基础上，新增一个系统级悬浮按钮。用户开启后台待命后，应用继续显示原通知，同时在其他应用上方显示悬浮按钮；点击按钮后直接打开扫码界面，扫码、复制、返回逻辑保持不变。

## 用户流程

1. 用户在主界面开启“后台待命通知”。
2. 如果系统悬浮窗权限未开启，应用提示用户前往系统设置授权。
3. 权限开启后，后台服务显示通知，并创建一个小型悬浮按钮。
4. 用户点击悬浮按钮，应用通过现有临时启动 Intent 打开 `MainActivity`，直接进入扫码界面。
5. 用户关闭后台待命时，通知和悬浮按钮都移除。

## 架构

复用现有 `StandbyService` 作为悬浮按钮宿主，避免新增独立后台服务。`StandbyService` 负责前台通知和悬浮按钮生命周期：启动时创建通知和悬浮按钮，销毁时移除悬浮按钮。

`MainActivity` 继续负责权限、扫码界面和设置开关。悬浮窗权限属于可选增强能力：没有权限时，通知入口仍按之前方式工作。

## 组件变更

- `AndroidManifest.xml` 增加 `SYSTEM_ALERT_WINDOW` 权限。
- `StandbyService` 增加 `WindowManager` 悬浮按钮创建、点击启动和移除逻辑。
- `MainActivity` 在开启后台待命时检查悬浮窗权限，缺失时跳转系统设置页。
- `strings.xml` 增加悬浮按钮文本和权限提示文案。

## 错误处理

如果没有悬浮窗权限，服务不创建悬浮按钮，但仍保留原有通知入口。这样权限被拒绝不会破坏现有功能。

## 测试

保留现有 Intent 单元测试。实现后运行 Gradle 测试；如果本地环境无法完成 Android 构建，则至少运行可用的 Gradle 验证命令并记录失败原因。
